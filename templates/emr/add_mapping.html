About Data Mappings

Data mappings define how fields in our system correspond to fields in the external system. They enable accurate data translation during synchronization.

Transformation Rules

Transformation rules are JSON objects that specify how to convert data between systems. Here are some examples: 

{
  "type": "date_format",
  "format": "%Y-%m-%d"
}

Converts a date to the specified format

{
  "type": "mapping",
  "map": {
    "male": "M",
    "female": "F"
  },
  "default": "U"
}

Maps values from one set to another

{
  "type": "unit_conversion",
  "from": "mmol/L",
  "to": "mg/dL"
}

Converts between different units of measurement
